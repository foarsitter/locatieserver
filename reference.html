Reference¶

API methods of the locatieserver grouped in this module for clean imports.

locatieserver.client.free(q='*:*', fl='id,weergavenaam,type,score', sort='score desc, sortering asc, weergavenaam asc', df='tekst', rows=10, start=0, wt='json', indent=True, lat=None, lon=None, fq='type:(gemeente OR woonplaats OR weg OR postcode OR adres)')[source]¶

Free service

For more info: https://github.com/PDOK/locatieserver/wiki/API-Locatieserver#5vrije-geocoder-service

Parameters

  • q (Optional[str]) – Hiermee worden de zoektermen opgegeven. De Solr-syntax voor zoektermen kan hier worden toegepast, bijv. combineren met and, en het gebruik van dubbele quotes voor opeenvolgende zoektermen. Zoektermen mogen incompleet zijn. Ook wordt er gebruik gemaakt van synoniemen.

  • fl (Optional[str]) – Hiermee worden de velden opgegeven die teruggegeven dienen te worden.

  • sort (Optional[str]) – Hiermee kan worden opgegeven hoe de sortering plaatsvindt. De defaultwaarde is score desc, sortering asc, weergavenaam asc. Door voor deze string typesortering asc toe te voegen, kan de oude sortering worden gebruikt.

  • df (Optional[str]) – Met behulp van deze parameter kan aan bepaalde velden een extra boost worden meegegeven. Hiermee kan de scoreberekening worden aangepast. De defaultwaarde is exacte_match^1 suggest^0.5 huisnummer^0.5 huisletter^0.5 huisnummertoevoeging^0.5. Om alleen van het suggest-veld gebruik te maken, kan qf=suggest worden meegegeven.

  • rows (Optional[int]) – Hiermee wordt opgegeven wat het maximale aantal rijen (ofwel resultaten) is dat teruggegeven moet worden op deze bevraging.

  • start (Optional[int]) – Hiermee wordt opgegeven wat de index is van het eerste resultaat dat teruggegeven wordt. Dit is zero-based. In combinatie met de rows-parameter kunnen deze services gepagineerd worden bevraagd.

  • wt (Optional[str]) – Hiermee wordt opgegeven wat het outputformaat is van de bevraging.

  • indent (Optional[bool]) – Hiermee kan worden opgegeven of de teruggegeven JSON ingesprongen (geïndenteerd) moet worden.

  • lat (Optional[float]) – Werkt alleen in combinatie met lon. Hiermee kan een coördinaat (in lat/lon, ofwel WGS84) worden opgegeven. Met behulp van deze parameters worden de gevonden zoekresultaten gesorteerd op afstand van het meegegeven punt. Wanneer de locatie van de gebruiker bekend is, kan op deze manier effectiever worden gezocht. Het meegeven van een coördinaat is met name nuttig voor de suggest- en vrije geocoder-services. Hier worden meestal meerdere resultaten teruggegeven. Als decimaal scheidingsteken moet een punt worden opgegeven i.p.v. een komma.

  • lon (Optional[float]) – Werkt alleen in combinatie met lat. Hiermee kan een coördinaat (in lat/lon, ofwel WGS84) worden opgegeven. Met behulp van deze parameters worden de gevonden zoekresultaten gesorteerd op afstand van het meegegeven punt. Wanneer de locatie van de gebruiker bekend is, kan op deze manier effectiever worden gezocht. Het meegeven van een coördinaat is met name nuttig voor de suggest- en vrije geocoder-services. Hier worden meestal meerdere resultaten teruggegeven. Als decimaal scheidingsteken moet een punt worden opgegeven i.p.v. een komma.

  • fq (Optional[str]) – Hiermee kan een filter query worden opgegeven, bijv. fq=bron:BAG. Met fq=* kan de default filter query worden opgeheven.
